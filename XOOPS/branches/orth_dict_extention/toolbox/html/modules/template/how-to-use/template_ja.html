How to Use (穴あき用例対訳)

穴あき用例対訳機能では、用例対訳の一部に単語を選択可能な箇所（パラメータ）を含む用例を作成できます。

このページでは、下記の機能について説明します。

  1. 1. 用例対訳の入力
  2. 2. パラメータの設定
  3. 3. 穴あき用例対訳の検索
  4. 4. カテゴリの管理
  5. 5. インポート・エクスポート
1. 用例対訳の入力
以下の手順で、穴あき用例対訳集を作成し、用例対訳を入力できます。
  1. 穴あき用例対訳の画面右側にある「穴あき用例対訳集の作成」ボタンをクリックします。
  2. 穴あき用例対訳で使用したい言語にチェックを入れます。また、穴あき用例対訳集の名前を入力して、OKを押します。
  3. 作成された穴あき用例対訳の名前をクリックし、用例対訳の編集画面に移動します。
  4. 表の1行目に表示されたドロップダウンボックスで、入力したい言語を選択します。
  5. 「穴あき用例対訳の追加」ボタンを押し、新規に行が追加されます。
  6. 追加された行をクリックして、用例対訳を入力します。
2. パラメータの設定
次の手順で、用例中に使用するパラメータを設定することができます。
  1. パラメータの列をクリックしてポップアップを開きます。
  2. 「パラメータの追加」ボタンをクリックし、パラメータ一覧に行を追加します。パラメータはインデックスの数字で識別され、用例中には[0], [1] などの形式で挿入されます。
  3. パラメータのタイプを選択します。新規のパラメータを追加する場合は、さらに以下の手順を行います。
  4. 「パラメータの編集」ボタンを押して、「穴埋め単語集合の編集」ポップアップを開きます。「穴埋め単語集合の追加」を押して新たなパラメータを追加し、その名前を設定します。
  5. 各パラメータの横の「編集」ボタンを押し、「穴埋め単語の編集」ポップアップを開きます。「穴埋め単語の追加」ボタンを押して、パラメータで選択可能な単語を設定します。
デフォルトのパラメータとしては、以下が使用できます。
  • [month]: 月の数値、または名前
  • [date] : 日付を表す数値
  • [hour] : 時刻の時(hour)を表す数値
  • [minute] : 時刻の分(miniute)の数値
  • [number]: 任意の数値
  • [text]: 任意の文字列
3. 穴あき用例対訳の検索
穴あき用例対訳集の一覧、または穴あき用例対訳編集画面で「検索」リンクをクリックすることで、穴あき用例対訳の検索が行えます。
検索を実行すると、検索条件にマッチした用例が表示されます。このとき、用例のパラメータは、ドロップダウンボックスまたはテキストボックスで表示されます。穴埋め単語集合を設定したパラメータは、ドロップダウンボックスから設定した単語を選択できます。また、デフォルトのパラメータは、テキストボックスに値を入力できます。これらのパラメータを設定すると、もう一方の言語の用例にも適切な語が設定されます。
4. カテゴリの管理
穴あき用例対訳の属するカテゴリを設定できます。
  1. カテゴリの列をクリックしてポップアップを開きます。
  2. 設定したいカテゴリにチェックを入れます。
  3. 適切なカテゴリがない場合には、「カテゴリの編集」リンクをクリックします。「カテゴリの追加」ボタンを押すと、新しいカテゴリを追加できます。
5. インポート・エクスポート
穴あき用例対訳集一覧で「インポート」ボタンを押すことで、ローカルのコンピュータのファイルから穴あき用例対訳集のインポートができます。また、各穴あき用例対訳集の行の右端にあるメニューから、「エクスポート」を選択することで、ローカルのコンピュータへ穴あき用例対訳集をエクスポートできます。インポート・エクスポートで使用するファイルは、以下のような形式を持つタブ区切りのテキストファイルとなります。
以下に、インポート・エクスポートで用いるテキストファイルの形式の例を示します。
idtypecat/parjaen
1parallel_text20,21<param id="0" type="float" />個より多くの<param id="1" domains="41" />を採ってはいけません。Picking more than <param id="0" type="float" /> <param id="1" domains="41" /> is prohibited.
2parallel_text20この付近には<param id="0" domains="40" />が多く生息しています。Many <param id="0" domains="40" /> inhabit in this Neighborhood.
... (省略)
20category案内Guide
21categoryルールRule
... (省略)
40parameter動物animal
41parameter植物plant
... (省略)
60word40ナマケモノsloth
61word40ゴリラgorilla
62word41百合lily
1行目は項目名です。2行目以降の各行には、用例・カテゴリのいずれかが記載されます。各列に記述する内容は以下の通りです。
  1. ・1列目: 各行に記述されるデータのID
  2. ・2列目: 各行に記述されるデータの種類。（parallel_text, category, parameter, wordのいずれか）
    • parallel_text: 用例対訳文の記述。cat/parに、属するカテゴリを記載。
    • category: カテゴリ名の多言語での記述
    • parameter: パラメータ名の多言語での記述
    • word: 穴埋め単語の多言語での記述。cat/parに、属するパラメータを記載。
  3. ・3列目: 用例の属するカテゴリ、または穴埋め単語が属するパラメータのIDを記載。
  4. ・4列目以降: 各言語での表現（1行目の項目名に記述される言語コードについては、ISO 639の言語コードを参照して下さい）
パラメータ部分は、"param"というタグに記載されます。"id"属性に、パラメータのインデックスが設定されます。また、パラメータのタイプがtype属性に以下の値が設定されます。ただし、言語グリッドが利用するサービスが扱うサービスとの互換性のため、一部の属性は、エクスポート時に以下のように設定されます。
  • date -> day_of_month
  • number -> float
